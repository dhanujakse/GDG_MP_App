==================================================
DESIGN PHILOSOPHY
==================================================

This is NOT a social media application.

This is NOT a feature-rich super app.

The application should have ONLY the features required for citizens to report problems and for MPs to understand and solve them.

Every unnecessary button, menu, or screen should be removed.

The interface must be so simple that:
- A first-time smartphone user can use it.
- Elderly citizens can submit complaints easily.
- Users should never feel confused.
- Every screen should focus on one task only.

Maximum 5 bottom navigation items.

Maximum 1 primary action per screen.

Avoid hidden menus and complex navigation.

==================================================
COMMUNITY COMPLAINT JOINING
==================================================

Instead of creating duplicate complaints, nearby citizens should be able to JOIN an existing complaint.

Example

Citizen A

Reports:

"No drinking water."

↓

Citizen B opens the app.

↓

The app detects an existing nearby complaint.

↓

Show:

"This issue has already been reported."

Buttons:

[ Join Complaint ]

or

[ Create New ]

If the citizen joins:

- Their support is counted.
- Their GPS is added.
- Their photo can be uploaded as additional evidence.
- Their report increases the impact score.

Do NOT create duplicate complaints unnecessarily.

==================================================
DESIGN RULES
==================================================

Every screen should have:

One primary button.

Minimal text.

Large icons.

Large cards.

Large touch targets.

Rounded corners.

Simple language.

Use icons with labels.

Avoid technical words.

Avoid clutter.

The app should feel calm, trustworthy, and effortless to use.
